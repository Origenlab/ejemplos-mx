# Cómo usar esta plantilla

Este es un artículo de blog **DEMO** escrito en `.mdx`. El blog del Master System
vive siempre en archivos `.mdx` (regla D3): nunca como páginas `.astro` sueltas.

## El flujo en tres pasos

1. **Configura los datos** en `src/config/site.ts` (identidad, contacto, taxonomía).
2. **Agrega contenido** como archivos Markdown en `src/content/`.
3. **Compila** con `npm run build` y publica el resultado estático.

## Por qué MDX

MDX permite mezclar Markdown con componentes de Astro cuando un artículo necesita
algo interactivo. Para la mayoría de los textos, el Markdown plano es suficiente.

> Reemplaza este contenido por un artículo real cuando construyas el sitio del cliente.
